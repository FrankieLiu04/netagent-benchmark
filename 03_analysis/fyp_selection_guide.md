4. # 毕业设计导师方向分类与选题指南
   
   本指南基于 `02_markdown/` 目录下已转换的选题文档。以下判断依据各导师的选题列表及可获取的简介幻灯片推断得出。
   
   ## 1. 方向集群
   
   ### A. 理论、信息论与数学密集型研究
   
   - **AG**： 信息论不等式，`f`-散度不等式，自拟理论课题。
   - **CN**： 混合型。`CN1` 为应用型网页/应用；`CN2` 为纯信息论研究。
   - **CHZ**： 电力系统优化，交流潮流，面向数学的工程优化。
   - **HFY**： 网络编码，批量网络编码，隐写术。
   - **POV**： 自拟课题，涉及图模型、编码理论、量子信息、信息论、数据分析。
   
   最适合：
   - 数学基础扎实
   - 能熟练阅读论文、进行证明或进行大量仿真
   - 对研究或研究生学习感兴趣
   
   ### B. 网络、无线通信、通信与系统
   
   - **HFY**： 多路径网络，网络编码，信息隐藏。
   - **YBL**： 用于网络工程的LLM，协议研究，流量分析。
   - **YJZ**： 用于无线系统的强化学习，基于机器学习的无线电环境地图构建。
   - **XJL**： 多无人机协同，机器学习推荐系统中的缓存，边缘/5G/LLM服务自拟课题。
   - **HC1**： 面向连接机器人的跨层无线智能。
   - **CKC**： 物联网设备与移动目标定位。
   - **SCL2/SCL3**： LiFi传感/通信，用于车载系统的语义通信。
   
   最适合：
   - 喜欢仿真器、协议、性能指标、系统实验
   - 希望完成技术扎实但不一定以产品为中心的工作
   - 对通信、网络、底层系统或边缘AI感兴趣
   
   ### C. AI、LLM、多模态、视觉与智能体系统
   
   - **CTL**： 智能笔记，时间管理，推荐系统。
   - **WCLAU**： 本地部署LLM应用，LLM智能体应用，智能体漏洞分析。
   - **XYY**： MLLM遗忘学习，基于扩散的图像生成，VLM安全性，LLM文献调研智能体。
   - **OUYANGWANLI**： 3D空间推理，多模态科学数据，变化检测，用于细胞生物学的LLM。
   - **TFX**： 计算成像，创意图像编辑，基于基础模型的图像质量评估。
   - **SCL**： 医学图像重建，LiFi传感，语义通信数据集，AI成本降低，幻觉减少。
   - **KSY**： 多模态欺诈分析，老年护理AI，体育预测，投毒防御，RL游戏，游戏开发。
   - **PKM2**： AI驱动的服务/应用。
   - **ZYY1**： LLM智能体机器人感知。
   - **WJY**： 用于自动驾驶和无人机路径规划的世界模型。
   
   最适合：
   - 希望选择具有明确作品集价值的流行AI课题
   - 熟悉模型训练、评估或LLM/VLM工具使用
   - 喜欢构建演示、流水线和应用型ML系统
   
   ### D. 安全、隐私、密码学与区块链
   
   - **SMC**： 隐私保护AI，模糊匹配，可搜索加密，阈值钱包，区块链凭证，跨链交易，支付合规，AI/系统安全。
   - **PKM1/PKM3**： 匿名凭证与应用密码学。
   - **DIWEN_XUE**： 协议歧义性分析，浏览器指纹识别，电子邮件混淆测量研究。
   - **WCLAU1**： 开源智能体漏洞分析。
   - **YZ**： 加密货币合规，链上冻结，场外交易市场风险，欺诈模式分析。
   - **HFY3**： 隐写术与信息隐藏。
   
   最适合：
   - 喜欢威胁模型、安全测量、隐私保护设计、合规或区块链系统
   - 希望课题能成为具有强烈安全色彩的作品集亮点
   
   ### E. 机器人、自主系统、传感与BCI
   
   - **HC**： 连接机器人，用于机器人操作的多模态传感。
   - **WJY**： 通过世界模型实现自动驾驶和无人机路径规划。
   - **XJL1**： 多无人机协同。
   - **GLX2**： 基础设施辅助的自动驾驶感知融合。
   - **ZYY1**： 基于LLM智能体的机器人感知。
   - **YYC**： 基于EEG的BCI通信/应用。
   - **CKC1/CKC2**： 物联网设备，定位系统。
   
   最适合：
   - 对具身AI、机器人学、感知或人机交互感兴趣
   - 熟悉传感器、数据流水线或仿真密集型开发
   
   ### F. 移动应用、人机交互与社会影响应用
   
   - **CN1**： 中大校巴导航网页应用。
   - **TML**： 智能手机应用和智能手机游戏。
   - **CTL**： 具有ML功能的生产力软件。
   - **POV1**： 改善生活/社区导向的自拟项目。
   - **GLX1**： 老年人护理实时监控仪表盘。
   - **ZYY2**： 智能3D室内导航应用。
   - **YYC2**： 学生设计的BCI应用。
   
   最适合：
   - 希望有一个可用于演示日的实体原型
   - 相比抽象理论，更偏好产品构建
   - 重视影响力、可用性和展示质量

   ## 3. 按学生画像推荐
   
   ### 如果你想要最稳妥、最易管理、最适合演示的毕业设计
   
   首选：
   - **CN1**： 校园校巴导航网页应用
   - **CTL1 / CTL2 / CTL3**： 笔记、时间管理、推荐系统
   - **TML1 / TML2**： 智能手机应用或游戏
   - **ZYY2**： 智能3D室内导航应用
   - **GLX1**： 老年人护理实时监控仪表盘
   - **PKM2**： AI驱动的服务/应用
   
   原因：
   - 这些课题有明确的产品边界
   - 在开题、中期检查和最终演示中易于解释
   - 可以清晰地展示UI、工作流程、用户测试和系统架构
   
   ### 如果你想要强大的AI/LLM简历价值
   
   首选：
   - **XYY4**： 基于LLM的文献调研系统
   - **XYY3**： VLM安全保障/安全对齐
   - **XYY1**： MLLM的机器遗忘学习
   - **WCLAU2 / WCLAU3**： 本地部署LLM应用或LLM智能体应用
   - **HC2**： 用于机器人的多模态传感智能
   - **ZYY1**： LLM智能体机器人感知
   - **TFX3**： 通过基础模型进行图像编辑和质量评估
   - **SCL4 / SCL5**： AI成本降低或幻觉减少
   
   原因：
   - 这些课题与当前AI招聘和研究趋势高度契合
   - 能为实习或研究生申请提供良好的故事线
   - 其中几个可以成为视觉效果出色的演示
   
   ### 如果你想要强大的安全/密码学/隐私方向
   
   首选：
   - **SMC3**： 基于MongoDB的多客户端可搜索加密
   - **SMC8 / SMC9**： AI安全或系统安全应用
   - **SMC5 / SMC7**： 区块链凭证和支付合规
   - **PKM1**： 匿名凭证应用

   - 将技术深度与当代相关性相结合

   - **YJZ1 / YJZ2**

   - 如果你喜欢仿真器、实验和优化指标，这是很好的选择

   ## 5. 选择前的风险提示
   
   ### 如果你数学或理论能力较弱，请谨慎选择
   
   - **AG**
   - **CN2**
   - **CHZ1 / CHZ2**
   - **HFY2**
   - **XJL1 / XJL2**
   - 密码学比重较大的 **SMC** 课题
   
   这些并非不好的课题，但如果你主要想要一个直接的实现项目，则不是好选择。
   
   ### 如果你自我管理或范围控制能力较弱，请谨慎选择
   
   - **TML1 / TML2**
   - **POV1 / POV2**
   - **PKM2 / PKM3**
   - **WCLAU2 / WCLAU3**
   - 许多自拟课题
   
   如果你能自己尽早界定范围并坚持执行，这些可以成为优秀的项目。
   
   ### 如果你缺乏硬件/机器人/传感背景，请谨慎选择
   
   - **HC1 / HC2**
   - **ZYY1**
   - **YYC1 / YYC2**
   - **CKC1 / CKC2**
   - **GLX2**
   
   这些课题比平均水平更令人兴奋，但也带来更多的集成风险。
   
   ### 如果你不喜欢数据收集/评估工作，请谨慎选择
   
   - **KSY** 项目
   - **GLX1**
   - **YZ** 项目
   - **DIWEN_XUE** 项目
   
   这些课题通常需要数据集、实证分析或仔细的评估流水线，而不仅仅是模型编码。
   
   ## 6. 实用选择建议
   
   - 如果你的主要目标是**以可控风险获得高分**，优先考虑 `CN1`、`CTL`、`TML`、`ZYY2`、`GLX1`。
   - 如果你的主要目标是**AI实习/AI作品集**，优先考虑 `XYY`、`WCLAU`、`HC2`、`TFX`、`ZYY1`、`XYY4`。
   - 如果你的主要目标是**安全/密码学职业信号**，优先考虑 `SMC`、`PKM1`、`DIWEN_XUE`、`YZ`。
   - 如果你的主要目标是**研究/研究生准备**，优先考虑 `AG`、`CN2`、`SMC`、`HC`、`WJY`、`OUYANGWANLI`、`CHZ`。
   - 如果你**尚未**有可靠的队友，请避免那些强烈假设需要两人或小组合作的课题，尤其是几个 `KSY`、`SMC` 和 `YYC` 的选项。
   
   ## 7. 我为你提供的个人推荐策略
   
   在不了解你确切背景的情况下，最明智的策略是：
   
   1.  在你的前三选择中保留一个**稳妥的应用型选项**。
       推荐：`CN1`、`CTL3`、`ZYY2` 或 `GLX1`。
   
   2.  在你的前三选择中保留一个**高潜力、兴趣驱动的选项**。
       推荐：`XYY4`、`HC2`、`SMC3`、`YBL1` 或 `YYC1`。
   
   3.  避免仅仅因为听起来流行而选择某个课题。
       例如，具身AI、多模态LLM或隐私保护AI很有吸引力，但如果你的基础或计算资源不够强，可能会变得很痛苦。
   
   4.  优先选择那些课题描述已暗示明确交付成果的导师。
       具有明显产出（如应用、仪表盘、原型、基准测试或系统）的课题通常比宽泛的"研究和探索"课题更容易执行好。
